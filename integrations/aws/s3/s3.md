# AWS S3 Integration Guide

This guide provides a comprehensive overview of the AWS S3 integration in this project, covering everything from installation to advanced usage like direct Multer uploads.

---

## 1. Installation & Dependencies

To use the AWS S3 integration, ensure the following dependencies are installed:

```bash
npm install @aws-sdk/client-s3 @aws-sdk/s3-request-presigner multer-s3 uuid
npm install -D @types/multer @types/multer-s3
```

- `@aws-sdk/client-s3`: Official AWS SDK for S3 operations.
- `@aws-sdk/s3-request-presigner`: Used for generating presigned URLs (for secure uploads/downloads).
- `multer-s3`: Multer storage engine for AWS S3.

---

## 2. Environment Configuration

Add the following variables to your `.env` file:

```env
AWS_REGION=your-region
AWS_ACCESS_KEY_ID=your-access-key-id
AWS_SECRET_ACCESS_KEY=your-secret-access-key
AWS_S3_BUCKET_NAME=your-bucket-name
# Optional: If using a custom endpoint (e.g., MinIO or LocalStack)
AWS_URL=https://s3.amazonaws.com
```

## 3. File Breakdown

The S3 integration is located in `integrations/aws/`. Here is what each file does:

### Interfaces
- **`interfaces/s3.interface.ts`**: Defines TypeScript interfaces for file uploads (`UploadFile`), upload results (`S3UploadResult`), file streams (`FileStreamResult`), and metadata (`FileMetadata`). This ensures type safety across the integration.

### S3 Module
- **`s3/s3.service.ts`**: The core logic for S3 operations. It provides methods for:
    - `upload()`: Uploads a file buffer directly to S3.
    - `getSignedUrl()`: Generates a temporary URL for private file access.
    - `stream()`: Streams a file from S3 (useful for proxying downloads).
    - `getMetadata()`: Retrieves file info (size, content type) without downloading.
    - `delete()`: Removes a file from S3.
    - `getPresignedUploadUrl()`: Generates a URL that allows a client to upload directly to S3.

### Multer Integration
- **`s3/multer-engine.service.ts`**: Configures the `multer-s3` storage engine. It provides a static `getMulterOptions()` method that handles:
    - S3 client initialization.
    - Automatic content-type detection.
    - Key (filename) generation using UUIDs.
    - Metadata attachment (e.g., who uploaded the file).
- **`s3/dummy-multer-file-filter.decorator.ts`**: A utility to restrict file types (e.g., images only) and enforce size limits during Multer uploads.
- **`s3/multer-engine-usage.controller.ts`**: A reference implementation showing how to use `S3FileInterceptor` with `S3UploadService` to handle file uploads in a NestJS controller.

## 4. S3 Service Usage

### Direct Upload via Service
```typescript
const result = await s3Service.upload({
  buffer: fileBuffer,
  fileName: 'example.png',
  mimeType: 'image/png',
  folder: 'profiles' // Optional: sub-folder in bucket
});
console.log(result.url); // Access URL
```

### Presigned Upload URL (Frontend Direct Upload)
Recommended for large files to avoid taxing your backend server.
```typescript
const uploadUrl = await s3Service.getPresignedUploadUrl(
  'uploads/large-video.mp4',
  'video/mp4',
  3600 // Expiry in seconds
);
// Frontend can now perform a PUT request to this URL with the file content.
```

## 5. Direct Multer Upload to S3

The `S3UploadService` allows you to stream files directly to S3 without saving them to the local disk first. This is highly efficient for production environments.

### Implementation Pattern

To implement an endpoint that uploads directly to S3 via Multer:

1. **Import the Interceptor and Service**:
   You need the `S3FileInterceptor` and the `S3UploadService`.

2. **Configure the Controller Route**:
```typescript
@Post('upload')
@UseInterceptors(
  S3FileInterceptor("file", S3UploadService.getMulterOptions())
)
@ApiConsumes("multipart/form-data")
async uploadFile(@UploadedFile() file: Express.Multer.File) {
  // At this point, the file is already in S3!
  // 'file.key' contains the S3 object key.
  return {
    url: `https://${process.env.AWS_S3_BUCKET_NAME}.s3.amazonaws.com/${file.key}`,
    key: file.key,
    originalName: file.originalname,
    size: file.size,
  };
}
```

### Key Benefits:
- **Zero Disk Usage**: Files are streamed directly to S3.
- **Automatic Content Type**: Correctly sets the MIME type on S3.
- **Metadata Support**: Easily add metadata like `uploadedBy` using the `metadata` callback in `multer-engine.service.ts`.
- **Validation**: Integrates with the `imageOnly` filter to block unwanted files BEFORE they reach S3.

## 6. Optional Configurations

### Custom Keys
You can customize how files are named on S3 by modifying the `key` callback in `multer-engine.service.ts`:
```typescript
key: (req, file, cb) => {
  cb(null, `custom-prefix/${Date.now()}-${file.originalname}`);
}
```

### File Filtering
The `dummy-multer-file-filter.decorator.ts` provides a starting point for validation:
- Checks MIME type (e.g., `image/`).
- Checks file size (e.g., max 5MB).
- Throws `BadRequestException` on failure.
